How to Use Our Tools

Service Execution Analyzer

Step-by-step guide for using the Process Memory Scanner

Requirements: Windows 7/Server 2008+, Administrator privileges, .NET Framework 4.8

Installation:

  1. Download the executable file from the Tools section
  2. Place the file in a dedicated folder for forensic tools
  3. Right-click the executable and select "Run as administrator"
  4. Accept any security prompts from Windows Defender

Usage:

  1. Select the process you want to analyze from the dropdown list
  2. Click the "Scan Process" button to begin memory analysis
  3. Review the results showing file paths and their status (present or deleted)
  4. Use the color coding to quickly identify suspicious entries (red = deleted, green = present)
  5. Export results to a text file using the "Export" button for documentation
Note: For best results, run this tool immediately after identifying suspicious activity to capture relevant memory artifacts before they're cleared.

Registry Executions Analyzer

Comprehensive guide to using the Registry Executions tool

Installation:

  1. Download the executable from the Tools section
  2. Extract the ZIP file to your preferred location
  3. Install .NET Core 3.1 Runtime if not already installed
  4. Double-click the executable to launch the application

Usage:

  1. Upon launch, the tool will automatically scan common registry locations
  2. Use the filter options to narrow down results by registry key or value
  3. Right-click any entry to access context menu options
  4. Select "Open File Location" to navigate to the executable path
  5. Toggle between dark and light themes using the settings menu
  6. Export results to CSV format for further analysis
Tip: Focus on entries marked as "Deleted" as these often indicate attempts to hide malicious activity. Cross-reference with other evidence.

DMA Device Detector

Using the PowerShell-based DMA detection script

Installation:

  1. Download the PowerShell script from the Tools section
  2. Save the file with a .ps1 extension
  3. Open PowerShell as Administrator
  4. Set execution policy: Set-ExecutionPolicy RemoteSigned

Usage:

  1. Navigate to the directory containing the script
  2. Run the command: .\DMA.Catcher.ps1
  3. Review the list of detected DMA-capable devices
  4. Investigate any unknown devices using the provided research links
  5. Check the generated log file (DMA_Report_YYYYMMDD.txt) for detailed information
  6. For comprehensive analysis, run the tool before and after connecting new devices
Security Note: Always verify the integrity of PowerShell scripts before execution. Consider setting execution policy back to "Restricted" after use for security.

Timeline Explorer

Guide for visualizing timeline data

Installation:

  1. Download the ZIP file from the Tools section
  2. Extract the contents to your preferred location
  3. Ensure .NET 6.0 Runtime is installed
  4. Run the executable file to launch the application

Usage:

  1. Import timeline data from CSV or other supported formats
  2. Use the timeline view to navigate through chronological events
  3. Filter events by type, keywords, or time range
  4. Group events by time intervals for better analysis
  5. Export your findings to various formats for reporting
